# Beauty House — Frontend

Публичният сайт на салона: HTML + CSS + Vanilla JS, свързан с backend API-то.
Естетика: **нежен пастелен лукс** (Fraunces + Manrope), редакторски фото-рамки, мека анимирана
цветна „аура" на фона на hero-а и плавни scroll-анимации. Без emoji — навсякъде инлайн SVG икони
(`js/icons.js`).

## Страници

| Файл | Какво е |
|------|---------|
| `index.html` | Начало — hero, услуги, преглед на екипа, галерия, отзиви, CTA |
| `services.html` | Услуги (тегли каталога + „от X лв" от цените на специалистите) |
| `team.html` | Екип (всички активни служители) |
| `booking.html` | Запазване на час — 4 стъпки (специалист → услуга → дата/час → потвърждение) |
| `auth.html` | Вход / Регистрация (табове) |
| `account.html` | „Моите часове" — клиент вижда своите, служител — графика си |
| `contact.html` | Контакти + форма |

## Структура

```
BeautyHauseFrontend/
├── *.html
├── css/styles.css        -- дизайн система (токени, компоненти, анимации)
└── js/
    ├── config.js         -- адрес на API-то (API_BASE) + данни за салона
    ├── api.js            -- fetch wrapper + JWT
    ├── auth.js           -- сесия (localStorage) + навигация
    ├── icons.js          -- SVG икон система (замества emoji)
    ├── ui.js             -- меню, scroll-reveal, споделени компоненти
    ├── interactions.js   -- 3D tilt на карти, брояч на статистики, аура parallax, магнитни бутони
    ├── home.js / services.js / team.js / booking.js / account.js / auth-page.js
```

## Как да пуснеш

1. **Стартирай backend-а** (виж главния README). По подразбиране на `http://localhost:5198`.
2. **Провери `js/config.js`** → `API_BASE` да сочи към твоя backend адрес.
3. **Отвори сайта** с локален сървър (за да работят `fetch` заявките):
   - Във VS Code: разширението **Live Server** → десен клик на `index.html` → *Open with Live Server*.
   - Или през терминал в папката: `npx serve` / `python -m http.server`.

   > Не отваряй файловете директно с `file://` — браузърът блокира заявките към API-то.

## Свързаност с API-то

- Публично (без вход): списък услуги, екип, услуги на специалист, свободни часове.
- `POST /bookings` изисква **клиентски** профил — ако не си влязъл, booking страницата те праща към вход и после обратно.
- Токенът се пази в `localStorage` и се прикача автоматично към всяка заявка.

## Бележки

- Ако backend-ът не е пуснат, страниците показват любезни резервни съобщения вместо да „чупят".
- Всички движения уважават `prefers-reduced-motion`.
- CORS е разрешен в backend-а за разработка (в `Program.cs`, политика `frontend`). За продукция ограничи до реалния домейн.
